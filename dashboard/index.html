VULT ID: Technical Bridge

Protocolo de comunicacion asincrona para el motor de Biometria Facial.

1. Arquitectura del Puente (PostMessage)

Debido a las restricciones de seguridad del navegador (Same-Origin Policy), la herramienta opera dentro de un iframe aislado. El intercambio de informacion se realiza mediante PostMessage, un canal de mensajeria bidireccional seguro que permite cruzar dominios sin exponer la seguridad del cliente.

Como funciona?

  • El Padre (tu sitio) "despierta" a la herramienta enviando los datos iniciales (DUI o Configuracion).
  • El Iframe (VULTID) procesa la biometria y "notifica" al padre mediante eventos de exito o errores.
  • Se incluye un canal de RESIZE para garantizar que la interfaz siempre se ajuste al tamano del contenido interno.

2. Formato de Respuesta del Servidor (Payload)

Para que el flujo de Login sea procesado correctamente, el endpoint de autenticacion debe devolver un objeto con las siguientes llaves obligatorias:

// Estructura Obligatoria
{
  "id": "A62B904F-8C1D-D3M0-942E-7B5A0C3F91D2",
  "nombre_usuario": "aris_vexel",
  "nombres": "Aris Elenai",
  "apellidos": "Vexel Kaltreem",
  "custom": {
    // Cualquier llave adicional aparecera aqui
    "rol": "administrador",
    "sucursal": "Centro"
  }
}
Nota: Todas las propiedades adicionales enviadas por la API que no sean las 4 obligatorias se agruparan automaticamente dentro del nodo custom.

3. Plantilla Maestra de Integracion

Estructura base recomendada para implementar el consumo de la herramienta en cualquier entorno web:

HTML + JS 
<!-- A. Contenedor e Iframe -->
<iframe
    id="vultid-tool"
    src="[SERVER_URL]/[UUID_DE_HERRAMIENTA]"
    allow="camera *;"
    style="width: 100%; border: none; min-height: 250px; transition: height 0.3s;">
</iframe>

<script>
    const vultidFrame = document.getElementById('vultid-tool');
    const VULTID_ORIGIN = "[SERVER_URL]"; // Ej: "http://localhost:8105"

    // B. Activacion (Envio de Datos al Cargar)
    vultidFrame.onload = function() {
        vultidFrame.contentWindow.postMessage({
            type: 'DUI_DATA',
            payload: { dui: "000000000" }
        }, "*");
    };

    // C. Escucha de Respuestas (Bridge de Salida)
    window.addEventListener('message', function(event) {
        if (event.origin !== VULTID_ORIGIN) return;

        const { type, height, payload } = event.data;

        // 1. Redimensionado Automatico
        if (type === 'RESIZE') {
            vultidFrame.style.height = height + 'px';
        }

        // 2. Login Intranet (Reconocimiento Facial)
        if (type === 'FACE_RECOGNITION_SUCCESS') {
            console.log("Login Exitoso:", payload);
        }

        // 3. Login Test / Demo
        if (type === 'FACE_DEMOGNITION_SUCCESS') {
            console.log("Demo Exitoso:", payload);
        }

        // 4. Verificacion de DUI
        if (type === 'FACE_VERIFICATION_SUCCESS') {
            console.log("DUI Verificado:", payload);
        }
    }, false);
</script>

4. Diccionario de Eventos Estandar

Evento (type) Direccion Descripcion
DUI_DATA Padre → Iframe Activa la herramienta enviando el DUI para validacion o modo demo.
UPLOAD_PROFILE_SUCCESS Padre → Iframe Inyecta una foto de referencia para el modo Login Test (demo).
RESIZE Iframe → Padre Notifica la altura del DOM interno. height contiene el valor en pixeles.
FACE_RECOGNITION_SUCCESS Iframe → Padre Login Intranet: resultado al identificar un rostro en la base de datos.
FACE_DEMOGNITION_SUCCESS Iframe → Padre Login Test (demo): resultado al comparar contra la foto de referencia subida.
FACE_VERIFICATION_SUCCESS Iframe → Padre DUI Validation: resultado al verificar que el rostro coincide con el DUI procesado.
